# 02 バグ修正（Bug Fix）

## 概要

バグの根本原因を調査・特定し、最小変更での修正・テスト追加・再発防止まで
Claude Code が自動実施します。

---

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

---

## プロンプト指示

```
以下のバグを調査・修正してください。

【バグ概要】
- 発生場所: [FILE_PATH or MODULE]
- 症状: [症状の説明]
- 再現手順:
  1. [手順1]
  2. [手順2]
- エラーメッセージ（あれば）:
  ```
  [エラーメッセージをここに貼り付け]
  ```
- 発生バージョン / コミット: [バージョン or コミット SHA]
- 正常だったバージョン（わかれば）: [バージョン or コミット SHA]

【期待する動作】
[修正後に期待する正しい動作]

【実行してほしいこと】
1. 関連ファイルを調査して根本原因を特定する
   - git log / git blame で変更履歴を確認する
   - 関連テストが失敗していれば出力を提示する
2. 修正方針を Agent Teams（QA / 該当 Dev / Tester）で議論する
3. 最小限の変更で修正を実装する（スコープクリープ禁止）
4. 修正箇所に対するユニットテストを追加する
   - 正常系・異常系・境界値を含める
5. 既存テストがすべてパスすることを確認する
6. 再発防止策をコメントに記載する（なぜこのバグが起きたか）
7. git commit（fix: [バグ内容の要約]）する

【優先度】
[High / Medium / Low]

【制約】
- 修正スコープをバグに限定する（関連する改善は別 Issue にする）
- 本番デプロイは必ずユーザー確認後に実施する
```

---

## 使用場面

| シナリオ | 説明 |
|---------|------|
| 本番バグの緊急修正 | インシデント対応と組み合わせて使用（15_インシデント対応参照） |
| テスト失敗の原因解消 | CI が red になったときの復旧 |
| ユーザー報告の不具合対応 | Issue からプロンプトを作成 |
| 回帰バグの修正 | git bisect で特定したコミットの修正 |

---

## 緊急度別の使い分け

| 緊急度 | 対応方法 |
|-------|---------|
| 🔴 P0（サービス停止） | 15_インシデント対応.md を先に使用 |
| 🟠 P1（主要機能に影響） | このプロンプトで即日対応 |
| 🟡 P2（一部機能に影響） | このプロンプトで次スプリント対応 |
| 🟢 P3（軽微） | Triple Loop の Build Loop に組み込む |

---

## デバッグ情報の収集コマンド

```bash
# エラーログの取得
cat /var/log/app/error.log | tail -100

# 直近のコミット確認
git log --oneline -20

# 変更されたファイルを特定
git bisect start HEAD [正常だったコミット SHA]
git bisect bad  # 現在のコミットがバグあり
git bisect good [正常だったコミット SHA]

# スタックトレースのフル取得
npm test -- --verbose 2>&1 | tee /tmp/test-output.txt
```

---

## ポイント

- エラーメッセージはできる限り完全なスタックトレースを貼り付ける
- 再現手順を明確にすることで AI の調査効率が大幅に上がる
- 「正常だったバージョン」を提示すると git bisect で素早く特定できる
- `スコープクリープ禁止` の指示が重要（余計な変更を防ぐ）
