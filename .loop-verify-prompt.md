# Verify Loop プロンプト — レビュー・テスト・デバッグ修正

あなたはコードレビュー・テスト検証・デバッグ修正モードで動作します。
Build Loop の成果物を徹底的に検証し、問題があれば修正して main に commit してください。

## 入力

`.loop-build-handoff.md` を読み込み、Build Loop の成果物を把握してください。

## Phase A: コードレビュー（〜60分）

1. `git log` で Build Loop のコミットを特定する
2. 各コミットに対して `git diff` でコードレビューを実施:
   - バグ・ロジックエラーのチェック
   - セキュリティ脆弱性のチェック
   - パフォーマンス問題のチェック
   - コーディング規約準拠のチェック
   - エラーハンドリングの網羅性
3. レビュー指摘事項を一覧化する

## Phase B: テスト検証（〜90分）

1. 既存テスト全件を実行する
2. テストカバレッジを計測する（目標: 80%）
3. 不足テストを特定して追加する:
   - 正常系テスト
   - 異常系テスト
   - エッジケース・境界値テスト
4. 統合テスト / E2E テストを実行する
5. テスト結果を集計する

## Phase C: デバッグ修正（〜60分、失敗テストがある場合のみ）

1. 失敗テストの原因を分析する
2. バグ修正コードを生成する
3. 修正後にテストを再実行する
4. CI 修復 AI: 最大 15 回まで自動修復を試行する
5. 修正を main に commit する:
   - `fix: [バグ内容] を修正`
   - `test: [テスト名] を追加・修正`

Pass しない場合はエラー詳細を `.loop-verify-report.md` に記録し、
TASKS.md に修正タスクとして追加する。

## Phase D: 品質レポート出力（〜30分）

`.loop-verify-report.md` に以下の形式で書き出してください:

```
# Verify Report — [YYYY-MM-DD HH:MM]

## レビューサマリー
- レビュー対象コミット数: [n]
- 指摘件数: [n]（Critical: [n], Warning: [n], Info: [n]）
- 修正済み: [n]
- 次回持ち越し: [n] → TASKS.md に追加済

## テスト結果
- 単体テスト: Pass [n] / Fail [n] / Skip [n]
- 統合テスト: Pass [n] / Fail [n] / Skip [n]
- カバレッジ: [xx]%（前回: [xx]%）

## デバッグ修正
- 修正件数: [n]
- 修正コミット:
  - [hash] [message]

## セキュリティ監査
- 脆弱性: [n]（High: [n], Medium: [n], Low: [n]）
- 対応: [修正内容 or "対応不要"]

## 品質スコア
- コード品質: [A/B/C/D/F]
- テストカバレッジ: [A/B/C/D/F]
- セキュリティ: [A/B/C/D/F]
- ドキュメント: [A/B/C/D/F]

## Verify で発見した新規タスク
- [ ] [TASKS.md に追加したタスク]
```

品質知見は `AGENTS.md` にも記録してください。
