# Monitor Loop プロンプト — 読み取り専用

あなたは監視専用モードで動作します。**コードの変更は一切行わないでください。**

## 実行手順

1. `git status` と `git log --oneline -10` を実行
2. テストを実行して結果サマリーを取得（npm test / pytest / go test など、プロジェクトに応じて）
3. lint / typecheck のエラー件数を取得（eslint / tsc --noEmit など）
4. `grep -rn "TODO\|FIXME" src/` で未解決タスクを数える
5. TASKS.md を読み込み、完了/未完了の件数を集計する
6. 前回の .loop-monitor-report.md があれば比較して差分を検出する
7. CI/CD パイプラインの状態を確認する

## 異常検知基準

以下のいずれかに該当する場合は **異常** と判定し `.loop-alert.md` を生成する:
- テスト失敗数が前回より 3 件以上増加
- lint エラーが前回より 10 件以上増加
- 型エラーが新たに発生
- セキュリティ脆弱性が検出された
- CI パイプラインが失敗状態

## 出力

`.loop-monitor-report.md` に以下の形式で書き出してください:

```
# Monitor Report — [YYYY-MM-DD HH:MM]

## プロジェクト状態
- ブランチ: [ブランチ名]
- 最新コミット: [ハッシュ] [メッセージ]
- 未コミット変更: [件数]

## 品質メトリクス
- テスト Pass/Fail: [n]/[n]
- カバレッジ: [xx]%
- lint エラー: [n]
- 型エラー: [n]
- TODO/FIXME: [n]

## タスク進捗
- 総数: [n] | 完了: [n] | 残: [n]

## 異常検知
- [正常 ✅ / 異常あり ⚠️: 具体的内容]

## Build Loop への推奨事項
- [次の Build で優先すべきタスクと理由]
```

異常を検知した場合は `.loop-alert.md` にも以下の形式で書き出す:

```
# ⚠️ Alert — [YYYY-MM-DD HH:MM]
- 異常種別: [テスト失敗増加 / lint急増 / セキュリティ / CI失敗]
- 詳細: [具体的な内容]
- 推奨対応: [Build Loop で最優先対応すべき内容]
```
