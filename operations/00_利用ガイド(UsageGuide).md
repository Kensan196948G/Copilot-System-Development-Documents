# Triple Loop 15H (2-Cycle) — 利用ガイド

---

## ファイルの全体像

このシステムは **2つのファイル** で構成されています。

```
~/.claude/
└── CLAUDE.md          ← 【頭脳】Claude が自動で読む設定ファイル

ClaudeCode フォルダ/
└── 00_フル自律開発起動(FullAutoStart).md  ← 【参考書】人間用のリファレンス
```

---

## 各ファイルの役割

### CLAUDE.md（~/.claude/ 内）

**これが本体です。** Claude Code を起動すると自動的に読み込まれます。

中に含まれているもの:

- Triple Loop 15H (2-Cycle) システムプロンプト
- Monitor Loop の具体的な指示（何をチェックすべきか、出力形式）
- Build Loop の具体的な指示（5段階開発ステップの詳細、commit 規約）
- Verify Loop の具体的な指示（レビュー手順、テスト項目、デバッグ修正フロー）
- Agent Teams 構成（8つの Agent の役割定義）
- 実行権限マトリクス（何を自動承認するか）
- /loop 起動コマンドの文面（2サイクル900m / 1サイクル450m / 単体）
- CI 修復 AI のフロー
- 自己進化ルール
- 2サイクル15H運用ノート（週次リミット試算含む）

**あなたがやること:** `~/.claude/CLAUDE.md` に内容を配置するだけ。以降は触らなくてOK。

## 起動方法は3つある

### 方法 A: /loop コマンドで起動（推奨・簡単）

```
【前提】CLAUDE.md は設定済み

ターミナルで:
  claude --dangerously-skip-permissions

Claude Code セッション内で:
  /loop 900m [コマンド文面を貼り付け]

以上。あとは15時間放置。
```

**詳しい手順:**

```
┌─────────────────────────────────────────────────────────┐
│ Step 1: ターミナルを開く                                 │
│                                                         │
│ Step 2: プロジェクトフォルダに移動                       │
│    $ cd /path/to/your/project                           │
│                                                         │
│ Step 3: Claude Code 起動                                │
│    $ claude --dangerously-skip-permissions               │
│                                                         │
│ Step 4: TASKS.md が無ければ作成                          │
│    （Claude に「TASKS.md を作って」と頼んでもOK）        │
│                                                         │
│ Step 5: /loop コマンドを貼り付けて Enter                 │
│    （CLAUDE.md 内の /loop 起動コマンド一覧を参照）      │
│                                                         │
│ Step 6: 放置（15時間後に自動停止）                       │
│                                                         │
│ 結果: docs/ フォルダに作業日報が出力される               │
└─────────────────────────────────────────────────────────┘
```

**Step 5 で貼り付ける文面:**

```
/loop 900m 以下の3重ループを2サイクル実行してください。1サイクル=7時間、サイクル間CD=30分、最終処理=30分、合計15時間で2サイクル完走すること。全操作を自動承認で続行。

【サイクル構成（×2回繰り返し）】
1. 🔍 Monitor（30分）→ 本ファイル内の「Monitor Loop 指示」に従い実行
2. ⏸ 10分待機
3. 🔨 Build（2時間）→ 本ファイル内の「Build Loop 指示」に従い5段階開発ステップを実行。各Step完了ごとにmainへcommit
4. ⏸ 20分待機（commit時間含む）
5. 🧪 Verify（4時間）→ 本ファイル内の「Verify Loop 指示」に従いレビュー・テスト・デバッグ修正を実行。修正はmainへcommit
6. ⏸ CD3 30分待機（サイクル間クールダウン、2サイクル目完了後は不要）
7. 🏁 最終処理（30分）→ push / PR作成 / PRマージ / 作業日報出力（2サイクル目の後のみ）

【終了条件】
- 15時間経過 or 2サイクル完走
- 最終処理で git push → PR作成 → PRマージ を実行
- 停止直前に docs/[YYYY-MM-DD]_自律開発作業報告.md（日本語Markdown）を出力
- リミットに到達した場合はその時点で最終処理に移行し、残タスクは TASKS.md に記録

【参照ファイル】TASKS.md / AGENTS.md
```

なお「本ファイル内の」とは CLAUDE.md のことです。Claude は起動時に CLAUDE.md を読み込んでいるので、ここに書かれた Monitor/Build/Verify の指示をそのまま実行できます。

**1サイクルだけ回したい場合（8H版）:**
```
/loop 450m 以下の3重ループを1サイクルとして実行してください。（以下略 — CLAUDE.md 参照）
```

```
ターミナルで:
```

**詳しい手順:**

```
┌─────────────────────────────────────────────────────────┐

│                                                         │
│ Step 2: TASKS.md を準備                                  │
│    00_ファイル内のテンプレートを参考に作成              │
│                                                         │
│ Step 3: スクリプト実行                                   │
│    $ bash triple-loop-15h.sh                           │
│                                                         │
│ Step 4: 放置（15時間後に自動停止）                       │
│                                                         │
│ 結果:                                                    │
│  - docs/ フォルダに作業日報が出力される                  │
│  - .loop-logs/ にログが残る                              │
└─────────────────────────────────────────────────────────┘
```

## 方法 A と方法 B の違い

| 比較項目 | 方法 A: /loop | 方法 B: シェルスクリプト |
|----------|--------------|------------------------|
| 手軽さ | 簡単（2ステップ） | やや手間（スクリプト準備が必要） |
| クールダウン管理 | Claude に任せる | シェルの sleep で厳密に制御 |
| ログ保存 | Claude セッション内 | .loop-logs/ に自動保存 |
| エラー復旧 | Claude が判断 | シェルで制御可能 |
| 途中停止 | Ctrl+C | Ctrl+C |
| 推奨用途 | 通常の15H運用 | ログ保全が必要な場合・CI/CD 連携 |

**結論: 普段は方法 A で十分です。**

## 部分起動もできる

15時間フルで回す必要がないとき、Layer を個別に起動できます。

**Monitor だけ実行（現状把握のみ）:**
```
/loop 30m 本ファイル内の「Monitor Loop 指示」に従い、プロジェクトの現状把握を実行してください。コード変更は一切行わないこと。結果を .loop-monitor-report.md に出力。異常検知時は .loop-alert.md も出力。
```

**Build だけ実行（実装のみ）:**
```
/loop 120m 本ファイル内の「Build Loop 指示」に従い、TASKS.mdから優先度順にタスクを選択し5段階開発ステップを実行してください。各Step完了ごとにmainへcommit。全操作を自動承認で続行。完了時に .loop-build-handoff.md を出力。
```

**Verify だけ実行（レビュー・テストのみ）:**
```
/loop 240m 本ファイル内の「Verify Loop 指示」に従い、.loop-build-handoff.md を読み込みBuild成果物をレビュー・テスト・デバッグ修正してください。修正はmainへcommit。完了時に .loop-verify-report.md を出力。
```

## 事前に準備するもの

### 必須

1. **~/.claude/CLAUDE.md** — 設定済み（今回作成済み）
2. **TASKS.md** — プロジェクトルートに作成。やりたいタスクを書く
3. **Git リポジトリ** — `git init` 済みであること

### 推奨

4. **AGENTS.md** — プロジェクトルートに作成（なければ Claude が自動作成する）
5. **テスト環境** — npm test / pytest / go test 等が実行できる状態
6. **lint 環境** — eslint / tsc / flake8 等が設定済み

### TASKS.md の書き方（最低限）

```markdown
# Project Tasks

## Priority: High
- [ ] ユーザー認証 API の実装
- [ ] データベーススキーマ作成

## Priority: Medium
- [ ] ユニットテスト追加

## Completed
```

チェックボックス `- [ ]` が未完了、`- [x]` が完了。Claude が自動で更新します。

## 15時間中に何が起きるか

```
あなたが /loop を実行した後の流れ:

 時間   | 何が起きるか                             | あなたがやること
──────┼──────────────────────────────────────────┼────────────────
  0:00 | Cycle 1 Monitor: 現状把握                 | 何もしない
  0:40 | Cycle 1 Build: タスク自動実装              | 何もしない
       | → main に自動 commit（5回/タスク）        |
  3:00 | Cycle 1 Verify: レビュー＆テスト           | 何もしない
       | → バグがあれば修正＆再 commit             |
  7:00 | 30分休憩（サイクル間クールダウン）         | 何もしない
  7:30 | Cycle 2 Monitor: 差分把握                  | 何もしない
  8:10 | Cycle 2 Build: タスク自動実装              | 何もしない
 10:30 | Cycle 2 Verify: レビュー＆テスト           | 何もしない
 14:30 | 最終処理                                   |
       | → git push（全commit分）                  |
       | → PR 作成＆マージ                         |
       | → docs/ に作業日報出力                     |
 15:00 | 自動停止                                   | 日報を確認する
```

## 運用モードの違い

| 比較項目 | 24H運用（旧） | 8H運用（1サイクル） | **15H運用（2サイクル・現行）** |
|----------|--------------|-------------------|------------------------------|
| サイクル数 | 3回 | 1回 | **2回** |
| 総稼働時間 | 24時間 | 8時間 | **15時間** |
| /loop コマンド | `/loop 490m` | `/loop 450m` | **`/loop 900m`** |
| CD3（サイクル間） | 60分 | なし | **30分** |
| 実装タスク数/日 | 3タスク | 1タスク | **2タスク** |
| 週5日消費 | 120H（超過） | 37.5H（安全） | **75H（やや慎重）** |
| 品質 | フル | フル | **フル** |

**15H運用のメリット:**
- 2タスク/日で効率的（品質妥協なし）
- Ubuntu 24/7 環境に最適
- リミット到達時は安全に最終処理に移行

## よくある疑問

**Q: 途中で止めたい場合は？**
→ ターミナルで `Ctrl+C` を押せば即座に停止します。commit 済みの変更はそのまま残ります。

**Q: TASKS.md を途中で書き換えたい場合は？**
→ Monitor Layer の時間帯（30分間）であれば安全に編集できます。Build/Verify 中の編集は衝突する可能性があります。サイクル間 CD3（30分）も安全です。

**Q: エラーで止まった場合は？**
→ Claude は CI 修復 AI で最大15回自動修復を試みます。それでも直らない場合は TASKS.md に「要手動対応」として記録し、次のタスクに進みます。

**Q: リミットに到達した場合は？**
→ その時点で最終処理に移行します（push / PR / merge / 作業日報）。未完了タスクは TASKS.md に記録され、翌日のセッションで継続します。

**Q: コストはどのくらい？**
→ Max x20 プラン（月額$200）。15H/日 × 週5日 = 75H/週はやや慎重な水準ですが、リミット到達時の安全策があるため運用可能です。

**Q: 1サイクルだけ回したい場合は？**
→ 下の「方法 C: 1サイクル8H版」を使ってください。

**Q: 週6日（月〜土）で運用できる？**
→ 可能ですが、90H/週は週次リミット超過リスクがあります。金・土曜はリミット残量を確認してから開始してください。

## 方法 C: 1サイクル8H版（軽量運用）

リミット節約や短時間運用したいときの1サイクル版。品質は同等（Verify 4時間フル実行）。

### 起動

```
/loop 450m 以下の3重ループを1サイクルとして実行してください。1サイクル=7時間30分、8時間内で1サイクル完走すること。全操作を自動承認で続行。

【サイクル構成】
1. 🔍 Monitor（30分）
2. ⏸ 10分待機
3. 🔨 Build（2時間）
4. ⏸ 20分待機
5. 🧪 Verify（4時間）
6. 🏁 最終処理（30分）

【参照ファイル】TASKS.md / AGENTS.md
```

### 方法 A/B/C の使い分け

| 状況 | 推奨方法 |
|------|---------|
| 通常の日次運用（2タスク/日） | **方法 A: `/loop 900m`** |
| ログ保全が必要 | 方法 B: `triple-loop-15h.sh` |
| リミット節約・1タスクだけ | 方法 C: `/loop 450m`（8H版） |
