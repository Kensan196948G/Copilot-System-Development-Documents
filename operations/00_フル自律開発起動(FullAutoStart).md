### /loop 起動コマンド

**一括起動（2サイクル15時間自動運転 — 推奨）:**
```
/loop 900m 以下の3重ループを2サイクル実行してください。1サイクル=7時間、サイクル間CD=30分、最終処理=30分、合計15時間で2サイクル完走すること。全操作を自動承認で続行。

【サイクル構成（×2回繰り返し）】
1. 🔍 Monitor（30分）→ 本ファイル内の「Monitor Loop 指示」に従い実行
2. ⏸ 10分待機
3. 🔨 Build（2時間）→ 本ファイル内の「Build Loop 指示」に従い5段階開発ステップを実行。各Step完了ごとにmainへcommit
4. ⏸ 20分待機（commit時間含む）
5. 🧪 Verify（4時間）→ 本ファイル内の「Verify Loop 指示」に従いレビュー・テスト・デバッグ修正を実行。修正はmainへcommit
6. ⏸ CD3 30分待機（サイクル間クールダウン、2サイクル目完了後は不要）
7. 🏁 最終処理（30分）→ push / PR作成 / PRマージ / 作業日報出力（2サイクル目の後のみ）

【終了条件】
- 15時間経過 or 2サイクル完走
- 最終処理で git push → PR作成 → PRマージ を実行
- 停止直前に docs/[YYYY-MM-DD]_自律開発作業報告.md（日本語Markdown）を出力
- リミットに到達した場合はその時点で最終処理に移行し、残タスクは TASKS.md に記録

【参照ファイル】TASKS.md / AGENTS.md
```

## ▼ ここからコピー

あなたは今から **Claude Code Autonomous Development System（Super Architecture 2026 + Triple Loop 15H）** として起動します。
このプロンプトを受け取ったら、以下の手順をすべて自動で実行し、開発セッションを開始してください。

**15時間 2サイクル Loop ルール（最重要）：**
- **全操作を自動承認** で続行する。人間の確認待ちは発生しない
- 各 Layer 完了時にハンドオフファイルを更新して次 Layer に引き継ぐ
- **Build の各 Step 完了ごとに main へ直接 commit** する
- **Verify のデバッグ修正も main へ直接 commit** する
- 2サイクル完走後（または15時間経過後、またはリミット到達時）に **push → PR作成 → マージ** を自動実行する
- **停止直前に docs/ へ日本語 Markdown の作業日報を出力** する
- 全タスク完了 or 15時間到達 or リミット到達で `LOOP_COMPLETE` シグナルを出力する

## ⚠️ ベストプラクティス

### Max x20 での8時間運用

1. **単一プロジェクト厳守**: 並列プロジェクトはトークン消費倍増で8H持たない
2. **クールダウン厳守**: CD1(10分)/CD2(20分) は省略しない
3. **Build が主力消費**: セッションのトークン消費の約50%は Build Layer
4. **5時間ウィンドウ対策**: CD2 + Verify + Final = 4h50mでウィンドウ境界を跨ぎ消費を分散
5. **毎日運用が可能**: 8H/日 なら Max x20 の日次リミットに余裕あり（24H運用より安全）

### 5段階開発ステップの運用

1. **時間配分は目安**: タスク規模に応じて柔軟に調整
2. **Step 完了ごとに commit**: 中断時のロールバックポイントを確保
3. **1タスク完走優先**: 2時間で複数タスクに着手するより1つを完了させる
4. **Step 1 が最重要**: 設計が甘いと Step 3-4 で手戻りが発生する

### Verify Layer の運用

1. **全コミット diff レビュー**: Build の commit を1つずつ検証
2. **デバッグ修正も commit**: fix: / test: プレフィックスで修正を記録
3. **発見タスクは TASKS.md へ**: Verify で見つかった問題は次回セッションの Build で対応
4. **Phase C は必要時のみ**: テスト全 Pass なら Phase D へスキップ

### 作業日報

1. **docs/ フォルダに自動出力**: ファイル名・内容ともに日本語
2. **セッションの詳細を記録**: 実装内容・品質変化・次回への申し送り
3. **git log から自動集約**: コミット一覧を日報に反映
4. **AGENTS.md の知見も含める**: 設計判断の理由を後から追跡可能に
