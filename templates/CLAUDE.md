## セッション開始時の確認事項

Copilotがこのプロジェクトで作業を開始する際は、以下を確認してください:

1. `TASKS.md` を読んで現在のPendingタスクを把握する
2. `AGENTS.md` のコーディング規約を確認する
3. `git status` で現在の変更状態を確認する
4. 作業開始前に「何をするか」を1文で宣言する

## Triple Loop 設定

### Monitor Loop（GitHub状態監視）
```
実行頻度: 作業開始時・各タスク完了後
確認内容:
  - オープンIssueの優先度確認
  - PR/CIステータスの確認
  - TASKS.mdの更新
```

### Build Loop（自律実装）
```
実装方針:
  - TASKS.mdのPending最上位から着手
  - AGENTS.mdの規約を厳守
  - 品質ゲートを全PASSするまで継続
  
品質ゲート（全PASSが必須）:
  !npm test
  !npm run lint
  !npm run typecheck
  カバレッジ ≥ 80%
```

### Verify Loop（品質検証）
```
検証手順:
  1. /review で変更差分をレビュー
  2. @security-agent でセキュリティチェック
  3. @test-writer でカバレッジ確認
  4. 問題なければコミット・PR作成
```

## 自律実行ルール

### 続行してよいケース
- テストが失敗 → 原因分析して自動修正
- lintエラー → 自動修正
- 型エラー → 自動修正
- コンパイルエラー → 自動修正

### 停止して報告すべきケース
- **セキュリティ上の懸念**（シークレット検出、認証バイパス等）
- **データ削除・破壊的変更**（DROP TABLE、大量削除等）
- **設計判断が必要**（アーキテクチャ変更、API仕様の破壊的変更）
- **同じ問題で3回連続失敗**
- **テスト失敗の原因が不明**

### 停止時のレポート形式
```
## 停止理由
[停止した理由を1行で説明]

## 現在の状態
[何をどこまで完了したか]

## 問題の詳細
[エラーメッセージ・スタックトレース等]

## 推奨アクション
[人間に対して何を決定・確認してほしいか]
```

## コンテキスト管理

- セッションが長くなったら `/compact` でコンテキスト圧縮
- 複雑なタスクは `/fleet` で並列サブエージェントに委譲
- 完了したタスクは都度 `TASKS.md` のステータスを更新
